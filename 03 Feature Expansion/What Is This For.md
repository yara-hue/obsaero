# 03 Feature Expansion

## Purpose

Generate the widest possible pool of _features, refinements, and delivery mechanisms_ that could be added to (or removed from) the locked core concept. This is where divergent thinking now lives — around the concept, not instead of it.

## Inputs

- Locked Core Concept ([[01 Understand Challenge/C - Locked Core Concept]])
- Challenge Brief ([[01 Understand Challenge/B - Official Challenge Brief]])
- Research findings ([[02 Research]])

## Process

1. Each member (Seif/Marwan/Yara) adds feature ideas to [[A - Feature Pool|Feature Pool]] under their section.
2. Every feature must state which brief focus area it serves and which constraint risks it carries (hardware? app? boarding delay?).
3. **Don't filter** — quantity first, judgment in Stage 04.
4. Features that strengthen the brief's weak spots (passenger-facing angle, storytelling, waste metrics) are especially valuable.

## Outputs

- A populated Feature Pool ready for scoring in Stage 04.
